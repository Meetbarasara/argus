runner, compose file that validates, git history started. Everything later builds on
this; nothing here should ever need revisiting.

**Read first:** 02 (repo layout, containers/ports), 05 (tooling & task definitions).
**Prerequisites:** Docker Desktop running; `uv` installed on host (`uv --version`).

- Recommend the folder be renamed `Agentic project` → `argus` (08 #1). If renaming is

## Steps

1. Step 0 above (rename decision, git init, first commit).
2. `uv init` shape per 02 layout; add dependencies; configure pyproject (ruff line 100,
   mypy per 05, pytest markers per 05, poe tasks per 05).
3. Write settings.py + trivial unit test.
4. Write docker-compose.yml (02 tables: services, ports, profiles, volumes,
   healthchecks) + the three Dockerfiles (platform/demoworld: copy pyproject+lock,
   `uv sync --frozen`, copy src; ui: nginx placeholder).
5. Copy config files + `.env.example` from 03. Create `.env` from example (keys blank ok).
6. Run the gate; update PROGRESS; commit.

## Acceptance criteria

- [ ] `uv run poe verify` green (lint, types, unit tests).
- [ ] `docker compose config` exits 0 (both profiles resolve; project name `argus`).
- [ ] `docker compose --profile platform up -d postgres redis` → both healthy.
- [ ] Platform + demoworld images build: `docker compose build api shopapi`.
- [ ] Git history: commit 1 = line-ending/ignore files; commit 2+ = `M00: ...`.

## Verification gate

```
$ uv run poe verify                    → all green, 1+ unit test passed
$ docker compose config --quiet        → exit 0
$ docker compose --profile platform up -d postgres redis
$ docker compose ps --format '{{.Service}} {{.Health}}'
                                       → postgres healthy, redis healthy
$ docker compose build api shopapi     → both images build successfully
```

**Out of scope:** any application logic, alembic (M02), UI app (M10).
